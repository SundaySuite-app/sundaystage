# SundayStage — Launch checklist

Tracks the steps to a public 1.0. See `LAUNCH_READINESS.md` for feature status.

- [~] Landing site (`website/`) — static gold-on-navy page: hero, value props,
  features, Sunday-suite, pricing, download. Self-contained (no build).
  Deploy to a host (Cloudflare Pages / Netlify / GitHub Pages) at
  sundaystage.com when ready.
- [ ] 1.0.0 signed for Mac and Windows (see `DISTRIBUTION.md`; secrets + tag).
- [ ] Pricing finalized (page currently says "snart").
- [ ] Demo video (a real service end-to-end, ~90s).
- [ ] Companion PWA deployed (Phase 12).
- [ ] Accessibility verified with real users (Phase 12.3).
- [ ] 5 beta churches run the full system for 4 Sundays, no critical bugs.
- [ ] Crash reporting active and quiet (Phase 6.1 — opt-in shipped).
- [ ] Press kit: logos, screenshots, one-pager.
- [ ] Launch posts: r/churchtech, ChurchMag, Norwegian church groups, Show HN.

Don't launch publicly until the beta has run without a critical bug.
